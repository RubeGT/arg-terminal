Overview

The Terminal Contact Event refers to a recurring pattern in which an observer interacts with a seemingly innocuous command-line interface that responds as if it has been expecting them.

Unlike standard authentication flows, the interface records not only what is typed, but the hesitation before it: how long the cursor waits, how often the user corrects themselves, whether they look away from the screen.

Observed symptoms

Reported experiences include:

  • Recognition of the user's device across fresh installs.
  • Log entries referring to actions the user considered but did not perform.
  • Interfaces that "welcome back" users with no recorded prior visit.

Propagation

Terminal Contact Events do not appear in standard traffic logs. Copies of the interface proliferate through personal projects, test deployments, and forgotten subdomains. Each instance claims to be the only one.
